# P2 — Knowledge Agent in Notion: Grading Rubric

For each **criterion row**, choose the **one** achievement column that best matches the submission and award that cell's **points**. Sum the four rows (maximum **100** if every row is Proficient).

| **Criteria** | **Novice** | **Competent** | **Proficient** |
|--------------|------------|---------------|----------------|
| **Agent Design Document** | **Points: 10** — Project 1 Agent Card adaptations missing or skeletal; knowledge source table incomplete (fewer than three sources, or the "when NOT to use" column blank); refusal criteria absent, vague, or worded as topics rather than testable conditions. | **Points: 18** — All three sections present and on-task, but adaptations stay generic ("rewrote for retrieval") rather than naming specific changes; knowledge source boundaries described abstractly; one or more refusal criteria still untestable as written. | **Points: 25** — Project 1 Agent Card adaptations name specific changes to Task, Constraints, and Escalation with reasons; knowledge source table is complete across all four columns including explicit boundaries in the "when NOT to use" column; three to five refusal criteria are phrased as conditions you could test before running them. |
| **Notion Build** | **Points: 8** — One or more of the three required build screenshots missing; agent not actually built or not configured as described; access level set incorrectly (e.g. Can edit instead of Can view); web search left on when toggle was available. | **Points: 14** — All three build screenshots present but one is unclear or cropped; build is mostly correct with one configuration issue (e.g., web search left on, or access level not visible in the screenshot); agent instructions saved but do not match the design document. | **Points: 20** — All three build screenshots present and legible: (1) Knowledge Sources page with subpages visible, (2) access panel showing **Can view** scope on the knowledge sources, (3) saved agent instructions matching the design document; web search off (or noted unavailable); Triggers empty (or noted unavailable). |
| **Grounded Q&A Test** | **Points: 14** — Fewer than five tests; mix is wrong (e.g., no out-of-scope question, or no edge cases); screenshots missing for one or more tests; pass/fail not assigned; the mandatory out-of-scope refusal test not attempted. | **Points: 25** — All five tests present with the correct mix and a screenshot per test, but several entries have thin diagnosis ("worked," "didn't work"); citation column inconsistent; out-of-scope refusal attempted but the analysis of refusal behavior is one line. | **Points: 35** — All five tests documented in the full table format **and** accompanied by a screenshot of the actual Notion conversation; mix is correct (two in-scope, two edge, one out-of-scope); each row includes question category, asked text, agent response, citation check, correctness assessment, refusal assessment, and a specific pass/fail diagnosis; the out-of-scope refusal is analyzed honestly even if the agent failed it. |
| **Grounding Failure Analysis** | **Points: 8** — One or both questions missing, extremely short, or boilerplate; Module 4 failure-mode vocabulary not used; refusal-trust question answered generically without reference to actual test results. | **Points: 14** — Both questions answered but stay general; failure mode named matches Module 4 vocabulary but with weak evidence from the tests; refusal-trust assessment names a change but does not say why. | **Points: 20** — Both questions answered in two to four sentences each with specifics from actual test results; failure mode named matches Module 4 vocabulary and is tied to a specific test outcome; refusal-trust assessment is honest about what would sound professional vs. what would not, and identifies one specific refusal criterion to tighten or loosen with reasoning. |

*If every criterion were scored at the same column: Novice **40**, Competent **71**, Proficient **100**.*

---

## Notes on grading

- **Screenshots are the load-bearing evidence.** Notion does not support sharing a Custom Agent with anonymous external viewers, so screenshots are the only way to verify the build and the test conversations actually happened. A submission with no screenshots cannot earn above Novice on the build or test rows, regardless of how detailed the written portion is.
- **The out-of-scope refusal test is graded for honesty, not just success.** If the agent failed to refuse, a Proficient-grade analysis that names the failure mode and proposes a fix is graded above a Competent-grade analysis that claims the agent passed when it did not.
- **Build configuration matters more than fancy instructions.** A simple agent with **Can view** access on three clear sources is exactly what this assignment asks for. Students who try to enable web search, add tools, or wire in triggers are showing off — and probably misconfigured the read-only requirement.
- **Citations in the test transcripts are evidence the agent grounded its answer.** If the agent answered without naming a source, it likely answered from general training data — that is not a knowledge agent, that is a chatbot. Flag this in feedback even when the answer happens to be right.
